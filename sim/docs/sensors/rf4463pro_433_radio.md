## Overview

The RF4463PRO is a 433 MHz SPI radio module built around the Silicon Labs Si4463 transceiver IC. It includes an integrated PA, LNA, and antenna matching network. On Apex it is the sole downlink for real-time telemetry — all sensor data, state machine status, and airbrake position are transmitted to the ground station over this link. It is receive-only during flight (no uplink commands are processed once the flight sequence is armed).

## HIL Simulation Notes

The RF4463PRO is **not simulated** in the HIL loop — the Teensy transmits real radio packets over the real hardware during HIL testing, as it would in flight. The laptop receives telemetry on a second RF4463PRO connected to the ground station PC (or a USB SDR dongle).

For pure SIL runs (no hardware), a software stub replaces the radio output with a local UDP broadcast or file log. This is configured in `config/hil.yaml` under `radio`.

| Config Field | Value | Notes |
|---|---|---|
| `mode` | `"hardware"` / `"udp"` / `"log"` | `hardware` = real RF link; `udp` = loopback for SIL |
| `udp_port` | 14550 | MAVLink-compatible default if using ground station software |
| `log_path` | `"logs/telemetry.bin"` | Binary log of all downlinked packets in SIL mode |

## Notes & Quirks

- **Si4463 configuration complexity:** The Si4463 chip is configured via a long sequence of radio configuration properties generated by Silicon Labs' [Wireless Development Suite (WDS)](https://www.silabs.com/developers/wireless-development-suite). Do not hand-edit the radio config registers — use WDS to generate the `radio_config.h` initialization array for the desired frequency, data rate, and modulation. The RF4463PRO module ships with a default config that may not match your desired parameters.

- **433 MHz in the US — check IREC rules:** 433 MHz is in the 70 cm amateur band in the US. Transmitting at +20 dBm on 433 MHz requires an amateur radio license (Technician class or above) unless operating under Part 15 power limits (~1 mW). Verify that your team has a licensed operator and that the transmission plan complies with IREC range safety requirements.

- **Antenna:** The RF4463PRO has a U.FL connector. A quarter-wave whip (~164 mm) is the simplest antenna and works well for a vertically-oriented rocket. Ensure the antenna is mounted away from metal structures and is not occluded by the airframe during flight. A helical or turnstile antenna on the ground station improves link margin.

- **SPI timing:** The Si4463 SPI interface has specific timing requirements around chip select assertion. Some SPI libraries issue CS too quickly after the last clock edge. If you see intermittent command failures, add a small delay before de-asserting CS or verify against the Si4463 SPI timing diagram.

- **CTS (Clear to Send) polling:** The Si4463 uses a CTS byte in its SPI response to signal that the previous command has been processed. Firmware must poll CTS before issuing the next command. Missing this step causes commands to be silently dropped — a common source of radio initialization failures.

- **TX current spike:** At +20 dBm the module draws ~85 mA, which is a significant spike on the 3.3 V rail. Ensure adequate decoupling capacitance (100 µF+ bulk, 100 nF ceramic) near the module's power pins to prevent rail sag that could brown out other sensors.

- **Packet structure:** Define a fixed-length telemetry packet structure and version it (include a 1-byte version field). This allows the ground station software to handle packet format changes across firmware versions without losing backward compatibility.
